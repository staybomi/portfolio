# 자기소개 홈페이지 PRD (Product Requirements Document)

## 1. 개요 (Overview)

채용담당자가 지원자의 자기소개와 포트폴리오를 한눈에 확인하고, 채용 적합 여부를 검증할 수 있도록 돕는 **개인 자기소개 홈페이지(포트폴리오 사이트)** 입니다.

| 항목 | 내용 |
| --- | --- |
| 제품명 | 자기소개 포트폴리오 홈페이지 |
| 문서 버전 | v1.0 |
| 작성일 | 2026-06-29 |
| 작성자 | (지원자 본인) |

---

## 2. 목표 (Goals)

- 채용담당자가 **짧은 시간 안에** 지원자의 역량과 경력을 파악할 수 있게 한다.
- 자기소개와 포트폴리오를 **시각적으로 매력적이고 인터랙티브하게** 전달한다.
- 채용담당자가 연락을 취할 수 있는 명확한 경로를 제공한다.

### 핵심 성공 지표 (Success Metrics)
- 채용담당자의 평균 체류 시간 증가
- 포트폴리오 항목 클릭/조회율
- 연락처(이메일/폼) 전환율(문의 발생률)

---

## 3. 대상 사용자 (Target User)

### 주 사용자: 채용담당자 (Recruiter / Hiring Manager)
- **목적**: 지원자의 자기소개와 포트폴리오를 보고 **채용 여부를 검증**하기 위함
- **특징**
    - 다수의 지원자를 검토하므로 **빠른 정보 파악**이 중요
    - 데스크톱·모바일 등 다양한 환경에서 접속 가능
    - 핵심 역량, 프로젝트 결과물, 연락 수단을 우선적으로 확인

---

## 4. 사용 목적 (Purpose)

> 채용담당자가 내 자기소개와 포트폴리오를 보고 **채용 여부를 검증**할 수 있도록 한다.

- 지원자의 강점, 기술 스택, 프로젝트 경험을 효과적으로 노출
- 신뢰감과 전문성을 주는 디자인/인터랙션으로 차별화
- 의사결정에 필요한 정보(이력, 성과, 연락처)를 빠짐없이 제공

---

## 5. 주요 기능 (Features)

### 5.1 메뉴(네비게이션) 구성

상단 고정 네비게이션 바(GNB)로 제공하며, 클릭 시 해당 섹션으로 **부드럽게 스크롤 이동**한다.

| 메뉴 | 설명 |
| --- | --- |
| **홈 (Home)** | 첫 화면. 핵심 한 줄 소개(헤드라인), 직군/포지션, 주요 키워드, CTA 버튼 노출 |
| **포트폴리오 (Portfolio)** | 프로젝트/작업물 목록을 카드 또는 그리드로 표시. 각 항목 클릭 시 상세(설명, 기술 스택, 결과, 링크) 확인 가능 |
| **자기소개 (About)** | 자기소개, 경력/학력, 보유 기술(스킬), 강점, 가치관 등을 소개 |
| **연락처 (Contact)** | 이메일, 전화, SNS/GitHub/LinkedIn 링크, 문의 폼 등 채용담당자가 연락할 수 있는 수단 제공 |

#### 요구사항
- 모든 페이지/디바이스에서 네비게이션이 **항상 접근 가능**해야 한다 (상단 고정).
- 현재 보고 있는 섹션이 네비게이션에 **활성 표시(active)** 되어야 한다.
- 모바일에서는 햄버거 메뉴 등으로 **반응형 대응**한다.

### 5.2 애니메이션 (Animation)

#### (1) 부드러운 스크롤 (Smooth Scroll)
- 네비게이션 메뉴 클릭 또는 페이지 스크롤 시 **자연스럽고 부드러운 스크롤** 경험 제공
- 섹션 진입 시 콘텐츠가 **페이드인/슬라이드인** 등으로 등장 (Scroll Reveal)
- 과도하지 않게, 정보 전달을 방해하지 않는 수준으로 적용

#### (2) 마우스 추적 잔상/파티클 효과 (Mouse Follow Trail / Particle)
- 마우스 커서를 따라다니는 **잔상(trail) 또는 파티클** 인터랙션 제공
- 사이트의 **개성·창의성·기술력**을 시각적으로 어필
- 성능 저하 없이 부드럽게 동작 (60fps 목표)

#### 요구사항
- 애니메이션은 **성능에 영향을 주지 않아야** 한다 (모바일 포함).
- 사용자가 원치 않을 경우를 대비해 `prefers-reduced-motion` 설정을 존중한다 (접근성).
- 마우스 파티클 효과는 **터치 기반 모바일 환경**에서는 비활성화 또는 대체 처리한다.

## 6. 화면 구성 (Page Structure)

단일 페이지(One-Page) 스크롤 구조를 기본으로 하며, 섹션 단위로 구성한다.

```
┌─────────────────────────────────────┐
│  [GNB]  홈 | 포트폴리오 | 자기소개 | 연락처  │ ← 상단 고정
├─────────────────────────────────────┤
│  # 홈 (Hero)                          │
│   - 헤드라인 / 포지션 / CTA            │
├─────────────────────────────────────┤
│  # 포트폴리오                          │
│   - 프로젝트 카드 그리드               │
├─────────────────────────────────────┤
│  # 자기소개                            │
│   - 소개 / 경력 / 스킬                 │
├─────────────────────────────────────┤
│  # 연락처                              │
│   - 이메일 / SNS / 문의 폼             │
└─────────────────────────────────────┘
   (전역) 마우스 추적 파티클/잔상 효과
```

## 7. 비기능 요구사항 (Non-Functional Requirements)

| 구분 | 요구사항 |
| --- | --- |
| **반응형** | 데스크톱 / 태블릿 / 모바일 모든 환경에서 정상 동작 |
| **성능** | 초기 로딩 빠르게, 애니메이션 60fps 유지 |
| **접근성** | 키보드 내비게이션, `prefers-reduced-motion` 지원, 적절한 대비/대체 텍스트 |
| **브라우저 호환성** | 최신 Chrome, Safari, Edge, Firefox 지원 |
| **SEO/공유** | 메타 태그, OG 태그(공유 시 미리보기) 설정 |
